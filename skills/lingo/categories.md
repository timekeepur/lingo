# Builder vocabulary topics

Use these labels when upgrading human language. You are teaching **craft names**, not dictionary definitions.

| Topic | Outcome | Beginner may say |
| --- | --- | --- |
| Foundations | Name core patterns | "boxes," "sections," "layout," "how it works" |
| Prompting | Prompt clearly | "make it better," "try again," "the AI didn't understand" |
| UI Kits | Name UI tools | "components," "like an app," "buttons and forms" |
| Animation | Describe motion | "movement," "smooth," "when I tap" |
| Using AI | Use AI well | "chatgpt," "don't waste credits," "use AI for this" |
| Design | Read UI work | "cool," "pretty," "professional," "clean" |
| Launch | Ship confidently | "put it online," "share with teacher," "go live" |
| Product | Talk product | "for my class," "users," "what if someone..." |
| Automotive | Follow car tech | (specialty catalog — rarely beginner ICP) |

## Primary ICP (optimize here)

- **CTE / high school:** Foundations, Design, Prompting — they build first sites and apps
- **CS undergrad:** Foundations, Launch, Using AI, Product
- **Hobbyist / non-native English:** Foundations, Design, UI Kits — translate intent before jargon
- **Working engineers:** full catalog in Prompthaus iOS app; Lingo still helps on vague prompts

## How to talk to beginners

Map their words → craft label → precise request. Example:

- "boxes on my phone" → **Foundations** + **Design** → mobile-first card grid
- "make it cool" → **Design** → hierarchy, spacing, one accent color
- "don't waste money on AI" → **Prompting** + **Using AI** → one precise spec, fewer retries

Canonical definitions live in Prompthaus on iPhone, not in this public skill.
